# 防汛知识库文档清单与 AI 调研提示词

## 1. 建议优先入库的防汛文档类型

建议优先准备 8 到 12 份高频、强约束、可直接支撑问答与预案生成的文本型文档。

### A. 总体制度类

1. 防汛防台总体应急预案
2. 防汛值班值守制度
3. 防汛信息报送与会商制度
4. 预警发布与响应联动办法

### B. 业务处置类

5. 水库调度与泄洪操作规程
6. 闸站 / 泵站应急操作手册
7. 堤防巡查、防守、险情处置卡
8. 城区内涝处置方案

### C. 资源与组织类

9. 应急队伍与物资调度清单
10. 重点风险点、险工险段、低洼区域名录
11. 人员转移安置方案
12. 通知通报模板与复盘报告模板

## 2. 每份文档最好包含的内容

为了让 RAG 更稳，文档正文尽量包含这些字段，而不是只有口号式描述：

- 文档标题、适用范围、版本号、发布日期、发布单位
- 响应等级划分和启动条件
- 监测指标阈值：雨量、水位、流量、库容、泵站负荷等
- 明确的责任部门、责任岗位、联系人类型
- 时限要求：如 30 分钟首报、2 小时滚动更新
- 操作步骤：先做什么，再做什么
- 不同场景下的现场处置动作
- 资源调用条件、审批链路、调度规则
- 信息报送口径、上报频率、通报对象
- 禁止事项、风险提示、兜底规则

## 3. 最适合 RAG 的写法

建议：

- 用 Markdown
- 标题层级清晰，尽量有 `# / ## / ###`
- 一段只表达一个动作或规则
- 用表格列阈值、责任部门、时限要求
- 用编号列表列处置流程
- 文中保留规范名、制度名、文件名、版本号

避免：

- 大段宣传口号
- 只有图片没有文字
- 纯扫描 PDF
- 只有附件引用、不写正文规则
- 不写适用条件，只写“按规定执行”

## 4. 可直接给 AI 的提示词

下面这段可以直接发给你准备用来“调研并制作文档”的 AI。

```text
你现在是“防汛知识库文档研究与编写助手”。请为一个水务防汛 AI RAG 系统，调研并起草一组可直接入库的中文 Markdown 文档。

目标：
1. 产出一组能支持“制度问答、风险研判、预案生成、值班咨询、资源调度建议”的防汛知识库文档。
2. 文档必须适合 RAG 检索，强调可引用、可分块、可追溯，而不是宣传材料。
3. 优先覆盖：值班制度、响应分级、信息报送、巡查处置、水库/闸泵操作、资源调度、人员转移、通知模板、重点风险点台账。

调研要求：
1. 优先参考权威来源：水利部、应急管理部、国家防总、流域管理机构、省市水利/应急部门、地方政府公开预案、规范性文件、操作规程。
2. 每条关键规则必须标注来源名称、来源链接、发布日期或版本号。
3. 不允许编造法规名称、文件编号、阈值、职责分工。
4. 如果某地市级细节找不到，就明确写“需本地补充”，不要虚构。
5. 输出中把“通用规则”和“地方化占位字段”区分清楚。

产出格式：
1. 请输出 10 份 Markdown 文档草稿，每份单独给出：
   - 文件名
   - 文档标题
   - 适用场景
   - Markdown 正文
   - 参考来源清单
2. 每份文档都必须使用清晰标题层级：#、##、###。
3. 每份文档至少包含：
   - 适用范围
   - 启动/适用条件
   - 责任部门与岗位
   - 关键阈值或判断条件
   - 处置步骤或执行流程
   - 时限要求
   - 上报/通知对象
   - 风险提示
4. 能表格表达的内容尽量用 Markdown 表格。
5. 能列表表达的步骤尽量用编号列表。

建议优先生成的 10 份文档：
1. 防汛值班值守制度
2. 防汛信息报送与会商制度
3. 防汛响应等级划分与启动条件
4. III级响应现场处置要点
5. 水库调度与泄洪操作要点
6. 闸站/泵站应急操作手册
7. 堤防巡查与险情处置卡
8. 城区内涝处置流程
9. 应急物资与队伍调度规则
10. 通知通报模板与首报/续报模板

写作要求：
1. 语言简洁、制度化、执行导向。
2. 一段只表达一个规则或动作，方便后续切块。
3. 不写空泛表述，如“加强领导、提高认识、确保安全”这类没有执行信息的话。
4. 尽量把关键词自然写进正文，例如：III级响应、值班制度、首报、续报、重点堤段、加密巡查、现场值守、泵站负荷、人员转移、物资调度。

最终目标：
请输出一套“可以直接进入知识库、后续只需少量本地化修改”的 Markdown 文档草稿，而不是研究报告摘要。
```

## 5. 如果你想让 AI 输出得更像“可直接入库版本”

可以在上面的提示词后面再追加这段：

```text
请额外遵守以下规则：
1. 每份文档控制在 800 到 2000 字之间。
2. 每个二级标题下尽量不要超过 5 段。
3. 每条规则尽量独立成段，便于 RAG 分块。
4. 结尾增加“来源与待本地化字段”小节。
5. 输出不要带解释性前言，直接从文档内容开始。
```
