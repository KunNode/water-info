# III级响应现场处置要点（通用版草案）

## 适用范围

本文件适用于本级防汛 III 级响应启动后，现场指挥、工程排险、群众转移、交通管控、信息报送等处置工作。地方公开预案通常将 III 级响应设置为“风险明显加大、重点工程和重点区域需要集中应对”的等级，要求副指挥或专班坐镇、加密监测、加强工程巡查抢修、组织受威胁群众转移、专业队集中备勤并滚动报送情况。

## 现场指挥链

| 层级 | 责任内容 |
|---|---|
| 本级指挥部 | 下达 III 级响应指令，明确重点区域、重点工程、重点时段 |
| 属地政府 | 负责现场组织、人员转移、安置点启用、属地管控 |
| 工程责任人 | 负责水库、堤防、闸泵、排水设施现场处置和技术复核 |
| 行业部门 | 负责交通、住建、城管、供电、通信、卫生等专业处置 |
| 抢险队伍 | 按指令集中备勤、机动增援、实施抢险 |

## 处置流程

接令到岗：收到 III 级响应通知后，立即建立现场工作群、明确总负责人与信息员、复核危险区清单。

加密监测：对雨量站、水位站、库区、堤段、闸泵、下穿隧道、地下空间、边坡和危旧房加密巡查。

工程排险：发现问题立即抢修、临时加固或封控，并严格执行既有调度方案和操作规程。

转移安置：按照“谁组织、转移谁、何时转、转何处、不擅返”闭环要求，对受威胁群众实施转移和安置。

交通与警戒：对积水路段、桥涵、堤外滩地、施工营地、危险岸线设置警戒、限行或封闭。

资源调度：专业抢险队集中备勤，物资、车辆、照明、抽排设备按重点区域前置。

滚动报送：对险情变化、转移人数、队伍到场、物资缺口和处置效果持续更新。

广州预案在 III 级响应中明确要求加强重点工程检查和应急加固、组织受影响群众转移并妥善安置、专业抢险分队集中备勤；水利系统对山洪防御则强调“五个关键环节”闭环落实。

## 风险提示

III 级响应最容易出现“工程处置和人员转移不同步”。凡是存在夜间强降雨、河道水位快速上涨、山区道路阻断、地下空间倒灌风险的区域，应把“先转移、后核损、严禁擅返”作为底线要求。不得因等待更精确数据而错失提前转移窗口。
