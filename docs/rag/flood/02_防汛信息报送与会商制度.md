# 防汛信息报送与会商制度（演示版）

## 一、目的

本制度用于规范防汛信息收集、核实、报送、会商、发布和归档工作，服务应急管理系统演示中的智能问答、预案生成和处置流程推理。制度强调“首报要快、续报要准、终报要全、会商要有结论、指令要闭环”。

## 二、信息类别

| 类别 | 主要内容 | 典型来源 |
|---|---|---|
| 雨情 | 降雨量、雨强、落区、未来预报 | 气象部门、雨量站、雷达外推 |
| 水情 | 河道水位、流量、涨落趋势、超警情况 | 水文部门、水位站、遥测平台 |
| 工情 | 水库、堤防、闸站、泵站、涵洞运行状态 | 工程管理单位、巡查人员 |
| 险情 | 管涌、渗漏、滑坡、漫顶、裂缝、闸门故障 | 属地政府、工程单位、现场队伍 |
| 灾情 | 受淹区域、受灾人口、转移人口、损失情况 | 应急部门、属地政府、行业部门 |
| 处置情况 | 队伍、物资、交通、抽排、封控、转移 | 成员单位、现场指挥部 |
| 舆情 | 媒体报道、网络关注、群众求助 | 宣传部门、热线平台、互联网监测 |

## 三、报送原则

1. **快速首报**：发生突发险情灾情后，先报告基本事实、初判风险和已采取措施，不等待所有数据齐备。
2. **滚动续报**：水位、雨量、险情范围、人员转移、抢险进度发生变化时，应及时续报。
3. **统一口径**：同一事件对上报送、横向通报和对外发布应使用一致的时间、地点、数据和风险判断。
4. **分级负责**：属地负责首报和现场核实，行业部门负责专业数据和技术判断，防指负责综合研判和调度指令。
5. **全程留痕**：电话、短信、平台、传真、会商纪要、签报、图片视频均应归档。

## 四、报送时限

| 场景 | 首报要求 | 续报要求 |
|---|---|---|
| 一般积水或小型工程异常 | 2 小时内报本级值班室 | 处置完成后报结果 |
| 河道超警、水库超汛限、泵站满负荷 | 1 小时内报本级防指 | 每 3 小时或有变化即报 |
| 堤防管涌、滑坡、漫顶风险 | 30 分钟内电话报告 | 每 1 小时滚动报告 |
| 人员被困、重大交通中断、大面积内涝 | 立即电话报告 | 实时更新救援和转移情况 |
| 重大险情灾情 | 立即电话报告，1 小时内书面首报 | 按上级要求加密报送 |

## 五、首报内容

首报应包含以下最小字段：

1. 事件名称和发生时间。
2. 事件地点，精确到乡镇、村社、工程或道路。
3. 事件类型，如超警水位、堤防渗漏、城区内涝、水库泄洪、人员被困。
4. 当前态势，包括水位、流量、雨量、积水深度、险情规模。
5. 初步影响，包括受威胁人口、道路、学校、医院、重要设施。
6. 已采取措施，包括巡查、封控、转移、抢险、抽排、调度。
7. 需协调事项，包括队伍、物资、专家、交通、电力、通信支持。
8. 联系人和联系电话。

## 六、续报内容

续报应突出“变化量”和“新动作”，不得简单重复首报。续报重点包括：

| 模块 | 说明 |
|---|---|
| 最新数据 | 雨量、水位、流量、库容、积水深度、泵站流量 |
| 风险变化 | 扩大、趋稳、减弱、转移、复燃等趋势 |
| 处置进展 | 队伍到场、物资到位、抽排进度、封控范围、转移人数 |
| 新问题 | 道路受阻、电力中断、通信不畅、群众安置压力 |
| 下一步 | 加密巡查、扩大转移、增派队伍、调整工程调度 |

## 七、会商启动条件

出现下列任一情形，应组织防汛会商：

1. 未来 24 小时预报有暴雨到大暴雨，且落区与前期强降雨重叠。
2. 主要河道水位接近或超过警戒水位。
3. 水库水位接近汛限水位，或需要预泄、错峰、泄洪。
4. 城区出现多点积水，交通、学校、医院、地下空间受影响。
5. 堤防、水库、闸站、泵站出现异常工情或险情。
6. 上级防指要求会商，或本级响应等级拟调整。

## 八、会商组织

会商由防指办公室牵头，视情况邀请气象、水文、水务、应急、自然资源、住建、城管、交通、公安、教育、卫生健康、电力、通信、属地政府和专家参加。会商可以采用现场会议、视频会议、电话会议或平台协同方式。

会商前由值班室准备基础材料：降雨实况与预报、水位过程线、水库库容、泵站运行、隐患点清单、重点保护对象、队伍物资状态、上一轮指令落实情况。会商后 30 分钟内形成纪要或调度指令。

## 九、会商结论格式

会商结论应包含：

1. 当前形势判断。
2. 未来 6 小时、12 小时、24 小时风险趋势。
3. 重点防守区域和重点工程。
4. 响应等级维持、启动、升级或终止建议。
5. 具体调度指令，包括责任单位、完成时限和反馈要求。
6. 信息发布和公众提醒口径。

## 十、信息发布要求

对外发布应坚持权威、准确、及时、适度原则。未经授权，任何值班人员不得以个人名义发布灾情、伤亡、工程险情、停课停工停运等敏感信息。涉及人员转移、交通管制、泄洪预警、停水停电等事项，应由防指或授权部门统一发布。

## 十一、报送质量检查

| 常见问题 | 修正要求 |
|---|---|
| 只有结论没有数据 | 补充雨量、水位、流量、积水深度等支撑信息 |
| 地点模糊 | 补充行政区、工程名称、桩号或坐标 |
| 措施笼统 | 写清责任单位、队伍人数、设备数量和完成时间 |
| 续报重复首报 | 突出新增变化和下一步措施 |
| 多口径冲突 | 明确采用经会商确认的统一口径 |

## 十二、演示用关键词

信息报送、首报、续报、终报、防汛会商、会商纪要、滚动报送、统一口径、雨情、水情、工情、险情、灾情、调度指令。
