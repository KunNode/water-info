# 防汛信息报送与会商制度（通用版草案）

## 适用范围

本制度用于规范防汛期间的信息采集、核实、首报、续报、终报以及会商组织流程，适用于雨情、水情、工情、险情、灾情、转移安置、应急响应和资源调度等事项。国家防总关于洪涝突发险情灾情报告的规定强调，报告遵循分级负责、及时快捷、真实全面原则；北京、广州等地方公开预案也普遍将气象、水务、应急、自然资源、交通、住建、城管等部门纳入联合研判和信息共享链条。

## 报送分类

| 类别 | 触发情形 | 报送重点 |
|---|---|---|
| 预警信息 | 暴雨、洪水、山洪、内涝、地灾预警发布、升级、解除 | 预警级别、影响范围、建议措施 |
| 工程险情 | 水库、堤防、涵闸、泵站、排水设施异常 | 位置、类型、发展趋势、处置措施 |
| 灾情信息 | 人员伤亡、受淹、停电停水停运、基础设施受损 | 人员、范围、影响、损失 |
| 转移安置 | 发布转移指令或形成集中安置 | 转移对象、人数、路线、安置点 |
| 响应行动 | 启动、升级、降级、终止响应 | 决定机关、等级、时间、行动要求 |
| 资源调度 | 队伍、物资、设备、车辆跨区调用 | 申请单位、数量、去向、到场时间 |

## 首报、续报、终报规则

首报以“先快后全”为原则，确认事件发生后立即报送已掌握的核心事实。

紧急情况下可先电话报送，再以书面形式补报。

续报以“滚动更新”为原则，持续报送险情变化、抢险进展、转移人数、受损扩展情况。

终报以“稳定结束”为节点，写明险情排除、响应终止、受灾损失、遗留问题和善后安排。

发生突发重大险情灾情时，县级以上防指应在 1 小时内向国家防总办公室报告；大江大河重要堤防、重要涵闸及大型和防洪重点中型水库重大险情应立即报告。

## 会商触发条件

建议至少在以下情形启动会商或升级会商：

预报未来 {{本地关键时段}} 内有强降雨、洪水、台风暴潮或山洪地灾复合风险。

监测指标达到或接近本地响应阈值，如雨量、水位、流量、库容、泵站负荷、重点道路积水深度。

工程出现渗漏、裂缝、管涌、设备故障、闸门失灵、泵站倒灌等异常。

需要提出群众转移、学校停课、道路封控、地下空间关闭、工程预泄或蓄泄联调建议。

需要启动、升级、降级、终止应急响应，或跨区调度队伍、物资、设备。

北京预案明确要求在重大灾害性天气来临时加强联合监测、会商和预报，并及时通报有关单位；广州预案要求响应期间收集汇总各成员单位落实情况和灾情信息。

## 首报要素模板

| 事项类型 | 最低必报字段 |
|---|---|
| 水库险情 | 工程名称、位置、时间、库水位、出入库流量、险情部位、下游影响、已采取措施 |
| 堤防险情 | 堤段名称、位置、水位流量、险情类型、影响范围、抢护力量、是否转移群众 |
| 涵闸/泵站险情 | 名称、位置、启闭或机组状态、河道水位、险情类型、是否影响排涝行洪 |
| 城区内涝 | 易涝点名称、积水范围、深度、受困人员、交通中断、抽排和封控措施 |
| 灾情 | 时间地点、灾害类型、死亡失踪、受困受淹、基础设施损毁、直接影响 |
| 转移安置 | 转移指令时间、危险区范围、人数、去向、安置点容量和保障情况 |

上述要素按《洪涝突发险情灾情报告暂行规定》的工程分类和灾情分类保留，便于后续 RAG 检索和模板复用。

## 风险提示

报送中最常见的问题是“数字先行、事实滞后”“部门各报各的、口径不统一”“只报损失、不报趋势”“只报处置、不报资源缺口”。会商纪要必须形成明确结论，包括风险判断、责任分工、时限要求、下次滚动会商时间和对外发布口径。
