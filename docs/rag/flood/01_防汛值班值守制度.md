# 防汛值班值守制度（演示版）

## 一、适用范围

本制度用于防汛应急管理系统演示场景，适用于市、区县防汛指挥机构办公室，水务、应急、气象、水文、住建、城管、交通、公安、电力、通信等成员单位，以及水库、堤防、闸站、泵站等工程运行管理单位。制度目标是确保汛期值班不断档、预警叫应不断线、险情处置不断链、信息报送有留痕。

值班状态分为常态值守、加强值守、应急响应值守和重大险情战时值守。常态值守以监测和信息接收为主；加强值守以预警叫应和重点部位盯防为主；应急响应值守以会商调度、抢险协调和转移避险为主；重大险情战时值守以现场处置、资源统筹和上级报送为主。

## 二、值班组织

| 岗位 | 配置要求 | 主要职责 |
|---|---:|---|
| 带班领导 | 每班 1 人 | 审定重要信息、决定会商调度、签发重要指令 |
| 值班长 | 每班 1 人 | 统筹当班任务、复核报送口径、组织交接班 |
| 信息值班员 | 每班 2 人 | 接收电话、平台告警、传真、专报和基层信息 |
| 技术支撑岗 | 每班 1 至 2 人 | 分析雨情、水情、工情、泵站负荷和预报成果 |
| 通信保障岗 | 每班 1 人 | 保障值班电话、视频会商、短信平台、专网和应急电源 |
| 联络协调岗 | 响应期增设 | 对接属地政府、成员单位、抢险队伍和物资仓库 |

启动 III 级及以上响应时，气象、水文、住建、城管、交通、公安、电力、通信等单位应按本级预案进驻或远程联合值守。发生重大工程险情、人员转移、城区大面积内涝、交通中断等情况时，可临时增设现场联络员和舆情联络员。

## 三、值班纪律

1. 值班人员必须按时到岗，严禁脱岗、漏岗、酒后值班或由无授权人员代班。
2. 电话、对讲机、移动终端、专网账号、视频会商设备必须保持可用，重要设备每班至少测试一次。
3. 重要信息必须执行“即收、即登、即核、即报、即办、即跟踪”，不得以“还在核实”为由压报、缓报。
4. 接到群众求助、基层险情、工程异常、媒体关注事项后，应先登记事件编号，再按类别转办，不得只转微信群不进台账。
5. 值班日志、电话记录、告警截图、会商纪要、调度指令、处置反馈、照片视频等资料应统一归档。

## 四、值班工作流程

### 1. 信息接收

信息来源包括监测平台自动告警、气象水文预警、工程管理单位报告、属地政府报告、群众来电、成员单位通报、现场工作组反馈、上级督办和媒体线索。值班员接收后应记录时间、地点、来源、事件类型、影响对象、初步风险、已采取措施、联系人和联系方式。

### 2. 初步分级

| 类别 | 判定示例 | 处置方式 |
|---|---|---|
| 咨询类 | 查询雨量、水位、道路积水信息 | 答复或转相关单位 |
| 一般事项 | 小范围积水、设备短时故障、普通投诉 | 登记转办，跟踪结果 |
| 重要事项 | 水位接近警戒、泵站超负荷、堤防渗水、人员受威胁 | 报值班长和带班领导，启动核实 |
| 突发事件 | 漫堤、管涌、滑坡、溃口、多人被困、大面积内涝 | 电话先报，立即会商调度，书面补报 |

### 3. 核实与研判

重要信息至少通过两个渠道核实。可采用电话回访、视频巡查、监测数据比对、现场照片定位、无人机图像、属地确认、工程单位确认等方式。无法立即核实但可能影响人员安全的，应按“先避险、后核实”的原则办理。

### 4. 报告与转办

一般事项由值班长转办相关单位。重要事项报带班领导，并通知属地政府和责任部门。突发重大险情灾情应立即向本级防指负责人和上级防指报告，必要时同步通报应急、公安、交通、消防、电力、通信等单位。

### 5. 跟踪闭环

每个事件应形成“接收、研判、指令、反馈、复核、归档”的闭环。对抢险队伍、物资调拨、人员转移、交通管制、工程调度等事项，应明确责任单位、完成时限和反馈方式。

## 五、响应期加密要求

| 响应状态 | 值班要求 | 信息报送频次 |
|---|---|---|
| 蓝色预警或常态汛期 | 24 小时值班，重点监测强降雨落区 | 每日 1 次值班摘要 |
| IV 级响应 | 带班领导在岗或随时到岗，重点部位清单化盯防 | 每 6 小时滚动报送 |
| III 级响应 | 成员单位联合值守，组织会商和叫应 | 每 3 小时滚动报送 |
| II 级响应 | 主要负责人加强调度，现场工作组下沉 | 每 2 小时滚动报送 |
| I 级响应 | 战时值守，抢险救援和转移避险优先 | 实时报告重大变化 |

## 六、叫应制度

当出现短历时强降雨、河道超警、水库接近汛限水位、闸站泵站异常、堤防险情、城区积水、山洪地质灾害风险、学校医院养老院受威胁等情况时，值班室应执行叫应。叫应对象包括属地政府负责人、工程责任人、巡查责任人、转移责任人、抢险队伍负责人和物资仓库联系人。

叫应记录应包含叫应时间、叫应人、被叫应单位、风险点名称、提醒内容、对方反馈、后续措施。连续三次无法接通的，应报告值班长并通过备用联系人、属地政府或现场人员补充叫应。

## 七、交接班要求

交接班必须面对面完成，重点交接以下事项：

1. 当前预警、响应等级和会商结论。
2. 超警河段、超汛限水库、高水位闸站、满负荷泵站。
3. 未办结险情、灾情、群众求助和上级督办。
4. 在途抢险队伍、已调拨物资、待确认转移人员。
5. 下一班需重点关注的雨区、水位涨势、交通风险和舆情风险。

交接后由交班人、接班人和值班长在值班日志中确认。

## 八、值班台账字段

| 字段 | 填写要求 |
|---|---|
| 事件编号 | 年月日 + 类别 + 序号 |
| 接报时间 | 精确到分钟 |
| 事件地点 | 区县、乡镇、村社、工程名称或道路名称 |
| 事件类型 | 雨情、水情、工情、险情、灾情、求助、舆情 |
| 风险等级 | 一般、较大、重大、特别重大 |
| 已采取措施 | 巡查、封控、转移、抽排、抢险、调度等 |
| 责任单位 | 主办单位和协办单位 |
| 当前状态 | 待核实、处置中、已控制、已办结、需上级支持 |
| 归档材料 | 通话记录、图片、视频、会商纪要、报文 |

## 九、常见问答

**问：监测平台告警和基层电话信息不一致时怎么办？**

答：先记录差异，再通过第三方数据、视频监控、现场责任人进行核实。涉及人员安全时，按较高风险等级先行叫应和避险。

**问：值班员能否直接发布公众提醒？**

答：不得擅自对外发布。公众提醒应按本级防指或宣传部门授权口径发布，紧急避险短信和广播通知按预案授权执行。

**问：重大险情尚未完全核实时能否上报？**

答：可以先电话报告“初步情况、正在核实、已采取措施”，随后补充书面首报和续报。

## 十、演示用关键词

防汛值班、领导带班、24 小时值守、联合值守、叫应、值班日志、交接班、险情登记、信息报送、闭环处置、应急响应、抢险调度。
