# 图5-4 告警实时展示页面截图文字描述

## 对应论文位置
第5章 `5.3.3 告警实时展示实现` 后。

## 建议图型
系统页面截图，建议使用“列表 + 连接状态 + 详情抽屉”同屏截图。

## 文字描述
截图应展示 `/warning/alarm` 页面。页面顶部是搜索栏，包含“告警等级”“状态”“时间范围”三个筛选条件，以及“搜索”“重置”按钮。中部主体是告警表格卡片，左上角标题为“告警列表”，右上角显示 WebSocket 连接状态标签，在线时为“实时连接”，断开时为“未连接”。表格列建议完整显示：站点、指标、等级、状态、告警信息、触发时间、确认人、操作。等级列使用彩色标签区分不同级别，状态列使用不同类型标签区分 `OPEN`、`ACK`、`CLOSED`。操作列应体现“确认”“关闭”“详情”按钮，其中“确认”“关闭”只对 `ADMIN`、`OPERATOR` 可见。若截图空间允许，建议同时打开右侧告警详情抽屉，展示站点编码、等级、状态、触发时间、最后触发、确认人、关闭人等信息，以表现页面的完整交互。

## 必含可视区域
- 搜索筛选区。
- 表格区。
- 右上角实时连接标签。
- 操作按钮。
- 可选的详情抽屉。

## 截图建议
- 优先选择已有一条 `OPEN` 告警和一条 `ACK` 告警的状态，便于体现状态流转。
- 若能复现实时推送，建议在论文说明中补一句“收到新告警后前端弹出提示并刷新列表”。
