# 图3-2 系统功能结构图文字描述

## 对应论文位置
第3章 `3.2.1 总体功能需求` 后。

## 建议图型
功能结构图或用例图，推荐使用树状功能分解图。

## 文字描述
图的中心是“智慧水务与防洪应急管理系统”。从中心向外分为三大分支。左侧为“业务平台功能”，下设站点管理、传感器管理、观测数据管理、阈值规则管理、告警管理、用户与角色管理、组织部门管理、审计日志、AI 代理接口。中间为“AI 智能决策功能”，下设自然语言查询、数据分析、风险评估、预案生成、资源调度、通知生成、执行监控、预案管理、会话查询。右侧为“前端展示与交互功能”，下设数据大盘、站点分布、告警列表、AI 智能指挥台、预案管理页、地图展示、大屏展示、系统管理页。图的底部再补一条“用户角色”横条，区分 `ADMIN`、`OPERATOR`、`VIEWER` 三类用户对功能的可见性和操作权限。

## 必含元素
- 三个一级模块：业务平台、AI 服务、前端管理端。
- 前端功能要体现 `/ai/command`、`/warning/alarm`、`/system/*` 等页面。
- 角色权限要体现系统管理页仅管理员可见，告警确认/关闭主要面向管理员和操作员。

## 标注建议
- 可在前端分支旁标注“统一交互入口”。
- 可在 AI 分支旁标注“多智能体协同”。
- 可在业务平台分支旁标注“统一业务治理”。
