C + D Optimization Roadmap

AI credibility and realtime reliability as one emergency-control loop

Proposed System Flow
1. Data

Stations + Alarms

Observation, threshold, alarm and sensor health signals.

2. Trigger

Risk Scan

Periodic and event-driven AI assessment jobs.

3. Reason

AI Evidence

Risk level, key drivers, plan excerpt and source trace.

4. Push

Realtime Bus

WebSocket + REST fallback + visible connection state.

5. Act

Bigscreen + Command

Alert on screen, expand evidence and execute plans in command view.

Scope Split

1

Phase 1: Realtime AI Assessment Loop

Connect bigscreen to existing AI assessment API/WebSocket, show source, freshness, risk, summary and fallback states.

2

Phase 2: Evidence and Plan Credibility

Expose key evidence, triggering station, plan excerpt, knowledge references and confidence markers in the command console.

3

Phase 3: Reliability Hardening

Add connection health, retry/backoff, stale-data badges, last successful sync and graceful degradation across both screens.
