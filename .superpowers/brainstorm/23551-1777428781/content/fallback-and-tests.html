Fallback and Acceptance Rules

The system must be honest about freshness and never show conflicting risk truth

Degradation States

A

Fresh AI Assessment

Use persisted latest assessment across all pages. Show source and assessed time.

B

Stale AI Assessment

Keep the last record visible, mark it stale, and avoid presenting it as current command truth.

C

Fetch or WebSocket Failure

Show connection/fetch status. Use cached last successful assessment only with offline label.

D

No AI Assessment

Show “暂无 AI 研判”. Alarm-derived hints are labeled as auxiliary, not AI risk.

Acceptance Tests
  • When latest persisted assessment is HIGH, both bigscreen and AI command show HIGH after refresh.
  • When WebSocket is disconnected, pages still recover latest assessment through REST.
  • When a new assessment is persisted, both pages update to the same id/assessedAt/level.
  • When no assessment exists, neither page invents an AI risk from alarms.
  • When assessment is stale, both pages show the same stale warning.
